AMX3_1 하이브리드 블록 구조 108B · 6.75 bpw

TriAttention 을 위한 이중 인코딩: WHT 경로(FA attention)와 polar 경로(스코어링) 병행.
Part크기필드역할
Part A
WHT		2 Bd_whtWHT 도메인 L2 norm (FA 디코더가 사용)
48 Bqs[48]3-bit Lloyd-Max (tbq3_1 동치, 128 elements)
Part B
Polar		2 Bd_rRayleigh σ (polar 스케일)
24 Bqr[24]3-bit Rayleigh r 인덱스 (64 페어)
32 Bqphi[32]4-bit uniform φ 인덱스

Part B → pre-RoPE (kxc, kyc) 직접 복원 → TriAttention 스코어링에 사용. 50% 유지 시 effective 3.375 bpw.

3-Kernel 스코어링 파이프라인

1. Raw score

grid=(n_kv, num_heads)
block=fc (thread/pair)
Part B 복원 + 17 offset sincosf 합 + block reduce

2. Z-normalize

grid=num_heads, block=256
head별 mean/std 계산
in-place 정규화

3. Aggregate

grid=(n_kv/T), block=128
per-slot max across all heads
→ out_scores[n_kv]

스코어링 공식: S_trig = Σ rel_r·cos(Δ·ω) - rel_i·sin(Δ·ω) (17 geometric Δ, 64 freq) + S_norm = max(0, qam-qma) · r

물리 eviction + attention sink

Histogram threshold

1024-bin 히스토그램 기반 K번째 largest. Single-thread kernel. n_kv=2500 기준 ~5μs.

Mask kernel

병렬 score ≥ threshold ? 1 : 0 비교.

Attention sink (C2)

mask[0..keep_first] = 1 강제 override.
BOS + 프롬프트 헤더 보호.

Physical eviction (C1)

mask=0 슬롯에 d_wht=0 AND d_r=0.
재스코어링 시 0점 → 영구 제외.

핵심 교훈

  • 모델 config의 key_length/value_length를 먼저 확인 — hidden/num_heads는 Q 기준, K 저장은 다를 수 있음 (gated attention, partial rotary)
  • Eviction 영구성은 Part A + Part B 모두 zeroing — 점수 경로(Part B)를 그대로 두면 재선택됨
  • Attention sink는 필수 — softmax 붕괴는 즉시 생성 품질 파괴 (무한 반복)
  • API는 처음부터 3단 설계 — 리소스 업로드/재사용/정리 분리가 자연스럽다. 단일 함수로 시작하면 리팩토링 비용 큼
  • make -j 는 target 종속성 자동 포함 안 됨 — 바이너리까지 재링크하려면 명시적 target 지정
  • 감사는 반복해도 새 발견 — self-review 본질적 한계. 시간 간격 + 다른 관점 = 최선
  • 피드백 메모리 실제 작동 — "No Fallbacks" 를 감사 중 위반했다가 사용자가 직접 지적해 원복
