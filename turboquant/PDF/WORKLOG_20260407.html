PPL 격차 조사 문제

현상

  • v1.5.1: PPL 509.9 (f16 419.8 대비 1.21x)
  • q4_0은 1.03x 격차 → TBQ3의 1.21x는 비트 수 대비 과도
  • 3-bit이 4-bit보다 왜 이렇게 나쁜가?

근본 원인

  • 비트 수가 아닌 스케일 팩터 수가 핵심
  • q4_0: 512차원당 스케일 16개
  • TBQ3: 512차원당 스케일 2개
  • 스케일 부족 → 양자화 노이즈 누적
핵심: TBQ3의 문제는 3-bit 자체가 아니라, 블록당 스케일 팩터가 적어서 softmax 분포가 평탄화되는 것. 스케일을 늘리는 대신 다른 접근이 필요.

Sub-block Scale 실험 실패

실험 설계

  • block_tbq3_0 구조체에 8개 서브스케일 추가
  • 32-element 그룹별 독립 스케일
  • 목표: 세밀한 스케일로 PPL 개선

결과

  • PPL 악화: 514.6 vs 기존 509.9
  • 서브스케일 자체가 양자화 노이즈 추가
  • WHT가 이미 분산 균등화 수행 중
교훈: WHT(Walsh-Hadamard Transform)는 이미 계수 분산을 균등화한다. 서브스케일은 중복이며 오히려 노이즈만 추가. → 스케일이 아닌 attention 보정으로 방향 전환.

Attention Sharpening 돌파구

이론적 배경

  • 3-bit 양자화 노이즈가 softmax 분포를 평탄화
  • MMSE 이론에서 유도: α = 1 + 1/(2×SQNR)
  • SQNR3bit ≈ 13.8 → α ≈ 1.036

장점

  • 메모리 비용 0 — 스칼라 곱셈 하나
  • 연산 비용 0 — softmax 전 단일 multiply
  • 이론적 근거 있는 보정 (경험적 해킹 아님)
결과: PPL 509.9 → 455.0 (1.21x → 1.08x). 메모리/연산 추가 비용 없이 PPL 격차 62% 감소.

V Rotation 버그 버그 수정

문제

  • attn_rot이 K와 V 모두에 적용
  • K: 안전 — Q도 같은 회전, 내적에서 상쇄
    (R·Q)·(R·K) = Q·K
  • V: 버그 — IWHT 디코드에 역회전 없음
  • V 출력이 회전된 상태로 나감

수정

  • attn_rot_v = false 설정
  • K만 회전 적용, V는 원래 값 유지
  • IWHT가 올바른 원본 V를 복원
  • PPL: 455.0 → 448.1 (1.08x → 1.07x)
원리: K 회전은 Q·K 내적에서 RTR = I로 상쇄되지만, V는 가중합 후 직접 출력되므로 회전이 남아있으면 결과가 왜곡된다.

TBQ3 / TBQP3 분리 α 최적화

TBQP3 (2-bit + QJL)

  • QJL 프로젝션이 attention 스코어에 추가 노이즈
  • 더 강한 sharpening 필요
  • α = 1.036

TBQ3 (3-bit)

  • 깨끗한 노이즈 프로파일
  • 가벼운 sharpening 충분
  • α = 1.016
결과: 단일 α 대비 분리 α 적용으로 tbqp3/tbq3 PPL: 473.7 → 454.7. 각 양자화 방식의 노이즈 특성에 맞는 보정.

Per-block Norm 개선

TBQ3 (3-bit)

  • Per-block norm 적용
  • 각 256-half 독립 정규화
  • 블록별 분산 차이를 정확히 포착

TBQP3 (2-bit + QJL)

  • Global norm 유지
  • QJL residual이 cross-block 512-WHT 사용
  • 블록 간 정보 공유 필요
설계 원칙: TBQ3는 256 블록 독립 → per-block norm이 자연스럽고, TBQP3는 512 전체 WHT → global norm이 올바른 선택.

tbq4_0 OOB 버그 수정 버그 수정

문제

  • vec_dot_fattn_vec_KQ_tbq4_0
  • D=512에서 K_tbq[0]만 읽음
  • 나머지 256 차원 미처리 → 잘못된 결과

수정

  • n_blocks = D/256 루프 적용
  • 모든 블록 순회하여 완전한 내적 계산
  • D=256, 512 모두 정상 동작
