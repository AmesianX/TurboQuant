PPL Gap Investigation Problem

Symptoms

  • v1.5.1: PPL 509.9 (vs f16 419.8 = 1.21x)
  • q4_0 only 1.03x gap → TBQ3's 1.21x is disproportionate
  • Why is 3-bit so much worse than 4-bit?

Root Cause

  • Not bit count, but scale factor count
  • q4_0: 16 scales per 512 dims
  • TBQ3: only 2 scales per 512 dims
  • Fewer scales → accumulated quantization noise
Key insight: TBQ3's problem is not 3-bit precision itself, but too few scale factors per block causing softmax distribution flattening. Instead of adding more scales, a different approach is needed.

Sub-block Scale Experiment Failed

Experiment Design

  • Added 8 sub-scales to block_tbq3_0 struct
  • 32-element groups with independent scales
  • Goal: finer-grained scaling to improve PPL

Results

  • PPL worsened: 514.6 vs baseline 509.9
  • Sub-scales themselves add quantization noise
  • WHT already performs variance uniformization
Lesson: WHT (Walsh-Hadamard Transform) already uniformizes coefficient variance. Sub-scales are redundant and only add noise. → Pivoted to attention correction approach.

Attention Sharpening Breakthrough

Theoretical Basis

  • 3-bit quantization noise flattens softmax distribution
  • Derived from MMSE theory: α = 1 + 1/(2×SQNR)
  • SQNR3bit ≈ 13.8 → α ≈ 1.036

Advantages

  • Zero memory cost — one scalar multiply
  • Zero compute cost — single multiply before softmax
  • Theoretically grounded correction (not empirical hack)
Result: PPL 509.9 → 455.0 (1.21x → 1.08x). 62% reduction in PPL gap with zero additional memory or compute cost.

V Rotation Bug Bugfix

Problem

  • attn_rot applied to both K and V
  • K: safe — Q gets same rotation, cancels in dot product
    (R·Q)·(R·K) = Q·K
  • V: BUG — IWHT decode has no inverse rotation
  • V output remains rotated → corrupted results

Fix

  • Set attn_rot_v = false
  • Only K gets rotation, V keeps original values
  • IWHT correctly recovers original V
  • PPL: 455.0 → 448.1 (1.08x → 1.07x)
Principle: K rotation cancels via RTR = I in the Q·K dot product, but V is directly output after weighted sum — any remaining rotation distorts the result.

Split α for TBQ3 / TBQP3 Optimization

TBQP3 (2-bit + QJL)

  • QJL projection adds extra noise to attention scores
  • Needs stronger sharpening
  • α = 1.036

TBQ3 (3-bit)

  • Cleaner noise profile
  • Lighter sharpening sufficient
  • α = 1.016
Result: Split α vs single α improved tbqp3/tbq3 PPL: 473.7 → 454.7. Each quantization method gets correction matched to its noise characteristics.

Per-block Norm Improvement

TBQ3 (3-bit)

  • Per-block norm applied
  • Each 256-half normalized independently
  • Captures per-block variance differences accurately

TBQP3 (2-bit + QJL)

  • Global norm retained
  • QJL residual uses cross-block 512-WHT
  • Requires information sharing across blocks
Design principle: TBQ3 uses independent 256-blocks → per-block norm is natural. TBQP3 uses full 512-WHT → global norm is the correct choice.

tbq4_0 OOB Bug Fix Bugfix

Problem

  • vec_dot_fattn_vec_KQ_tbq4_0
  • At D=512, only read K_tbq[0]
  • Remaining 256 dims unprocessed → incorrect results

Fix

  • Iterates over all blocks for complete dot product
  • Works correctly for both D=256 and D=512
