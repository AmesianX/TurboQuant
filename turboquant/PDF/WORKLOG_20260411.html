f16 Side Buffer 실험 35/35

양자화 전 원본 K를 f16으로 가로채서 별도 버퍼에 저장. attention이 양자화 대신 f16을 읽음.
  • Phase 1~3: WHT 도메인 f16 → raw f16 → generation 포함 → 수제 dot product
  • Phase 4~5: sharpening, *D/D 보상, Q WHT 스킵 → 효과 미미
  • Phase 6: f16 커널 redirect (fattn.tu에서 K→F16 override) → 거의 성공
  • Phase 7: 값 비교 디버깅 → src1 인덱스 누락 발견!
  • Phase 8: src1 적용 → 35/35 math bench, 멀티턴 성공

근본 원인: src1 인덱스 BUG

set-rows는 src1[i]로 KV 캐시 특정 위치에 저장.
side buffer는 순차(0,1,2...) 저장 → 위치 불일치.
컴포넌트수정 전수정 후
KV 캐시src1[0]=5 → position 5동일
Side buffertoken 0 → position 0 ✗token 0 → position 5 ✓
교훈: 추측보다 실측. f16 값과 side buffer 값을 직접 비교해서 발견.
