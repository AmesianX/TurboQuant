Root Cause Found Breakthrough

Cause 1: V IWHT Type Mismatch

  • Reduce stage: stored as half in __shared__ half KQ[]
  • IWHT stage: read as float via (float*)&KQ[0]
  • half→float type punning → precision loss
  • 512-point butterfly amplified error by √512 ≈ 22.6x

Cause 2: half2 VKQ Accumulation

  • V_DOT2_F32_F16_AVAILABLE macro
  • VKQ accumulator: half2 (10-bit mantissa)
  • Rounding error accumulates over hundreds of KV entries
  • IWHT amplifies this error
Key insight: With f16 KV, values already in f16 are accumulated in f16 — no precision loss. TBQ accumulates WHT-domain values then inverse-transforms via IWHT, so small errors get amplified — a fundamental difference.

Fix 1: V IWHT Float Staging Commit 30a4f48ae

Before (Bug)

  • Reduce loop wrote directly to half KQ[]
  • IWHT read via (float*)&KQ[0]
  • Type mismatch + possible race condition

After (Fix)

  • Store first in float register array tbq_regs[]
  • __syncthreads() barrier
  • Write as float to float *buf = (float*)&KQ[0]
  • Type consistency + synchronization guaranteed
D_V < nthreads edge case: float tbq_regs[D_V/nthreads] with D_V=64, nthreads=128 yields zero-size array → solved with constexpr int tbq_nregs = D_V >= nthreads ? D_V/nthreads : 1.

Fix 2: V_DOT2 Undef Commit 88bb214de

Pattern to force half2 → float accumulation path in TBQ V template instances: 

// Force float accumulation for TBQ V types

Affected Scope

  • VKQ accumulation: half2 → float2
  • KQ shared memory: half → float
  • Q_reg: half2 → float2
  • Q·K dot product also gains float precision

Application

  • Later: 67 more added (ALL_QUANTS)
  • Total: 70 TBQ V instances
  • f16 instances keep half2 (performance preserved)

Fix 3: Dynamic Sharpening Implementation Theory→Code

Before (Static)

  • α = 1 + 1/(2×SQNR)
  • Fixed constant (based on ctx=2048)
  • Same α for generation (N=10) and prefill (N=262K)

After (Dynamic)

  • α(N) = 1 + c × √(ln N)
  • N determined from runtime k_VKQ_max
  • Small α during generation, large α during prefill
  • No clamping needed (√ln grows ultra-slowly)
constexpr float c =
    (type_K == GGML_TYPE_TBQP3_0) ? 0.01304f :
    (type_K == GGML_TYPE_TBQ3_0)  ? 0.00579f :
    (type_K == GGML_TYPE_TBQP4_0) ? 0.00724f :
    (type_K == GGML_TYPE_TBQ4_0)  ? 0.00326f : 0.0f;
float alpha = 1.0f + c * sqrtf(logf(fmaxf((float)k_VKQ_max, 2.0f)));
sum *= alpha;
