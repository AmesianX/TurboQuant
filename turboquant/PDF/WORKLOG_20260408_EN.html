Root Cause Found Breakthrough

원인 1: V IWHT 타입 불일치

  • reduce 단계: __shared__ half KQ[]에 half로 저장
  • IWHT 단계: (float*)&KQ[0]으로 float 읽기
  • half→float 타입 펀닝 → Precision Loss
  • 512-point 버터플라이가 오차를 √512 ≈ 22.6배 증폭

원인 2: half2 VKQ 누적

  • V_DOT2_F32_F16_AVAILABLE 매크로
  • VKQ 누적기: half2 (10-bit 가수부)
  • 수백 KV 항목 누적 시 라운딩 오차 축적
  • IWHT가 이 오차를 증폭
Key: f16 KV에서는 이미 f16인 값을 f16 누적하므로 Precision Loss 없음. TBQ는 WHT 도메인 값을 누적한 뒤 IWHT 역변환하므로 작은 오차가 증폭되는 근본적 차이.

Fix 1: V IWHT Float 스테이징 Commit 30a4f48ae

수정 전 (버그)

  • reduce 루프가 half KQ[]에 직접 쓰기
  • IWHT가 (float*)&KQ[0]으로 읽기
  • 타입 불일치 + race condition 가능

수정 후

  • float 레지스터 배열 tbq_regs[]에 먼저 저장
  • __syncthreads() 배리어
  • float *buf = (float*)&KQ[0]에 float 쓰기
  • 타입 일관성 + 동기화 보장
D_V < nthreads 예외 처리: float tbq_regs[D_V/nthreads]에서 D_V=64, nthreads=128이면 크기 0 배열 → constexpr int tbq_nregs = D_V >= nthreads ? D_V/nthreads : 1로 Solution.

Fix 2: V_DOT2 Undef Commit 88bb214de

TBQ V 템플릿 인스턴스에서 half2 → float 누적 경로를 강제하는 패턴: 

// Force float accumulation for TBQ V types
#include "../common.cuh"          // defines V_DOT2_F32_F16_AVAILABLE
#undef V_DOT2_F32_F16_AVAILABLE   // force float path
#include "../fattn-vec.cuh"        // pragma once prevents re-define

영향 범위

  • VKQ 누적: half2 → float2
  • KQ 공유 메모리: half → float
  • Q_reg: half2 → float2
  • Q·K 내적도 float 정밀도 향상

적용

  • 초기: 3개 인스턴스 (TBQ_TUNING)
  • 이후: 67개 추가 (ALL_QUANTS)
  • 총 70개 TBQ V 인스턴스
  • f16 인스턴스는 half2 유지 (성능 보존)

수정 3: 동적 Sharpening 구현 이론→코드

이전 (정적)

  • α = 1 + 1/(2×SQNR)
  • 고정 상수 (ctx=2048 기준)
  • 생성 시(N=10)와 prefill(N=262K)에 동일 α

이후 (동적)

  • α(N) = 1 + c × √(ln N)
  • 런타임 k_VKQ_max로 N 결정
  • 생성 시 작은 α, prefill 시 큰 α
  • clamp 불필요 (√ln이 초느린 성장)
constexpr float c =
    (type_K == GGML_TYPE_TBQP3_0) ? 0.01304f :
    (type_K == GGML_TYPE_TBQ3_0)  ? 0.00579f :
    (type_K == GGML_TYPE_TBQP4_0) ? 0.00724f :
    (type_K == GGML_TYPE_TBQ4_0)  ? 0.00326f : 0.0f;
float alpha = 1.0f + c * sqrtf(logf(fmaxf((float)k_VKQ_max, 2.0f)));
sum *= alpha;
