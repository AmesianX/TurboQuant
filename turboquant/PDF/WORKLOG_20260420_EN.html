Part B → direct reconstruction of pre-RoPE (kxc, kyc) → fed to TriAttention scoring. At 50% retention, effective 3.375 bpw.

3-Kernel Scoring Pipeline

1. Raw score

grid=(n_kv, num_heads)
block=fc (thread/pair)
Part B reconstruct + 17 offset sincosf sum + block reduce

2. Z-normalize

grid=num_heads, block=256
Per-head mean/std
in-place normalization

3. Aggregate

grid=(n_kv/T), block=128
per-slot max across all heads
→ out_scores[n_kv]

Scoring formula: S_trig = Σ rel_r·cos(Δ·ω) - rel_i·sin(Δ·ω) (17 geometric Δ, 64 freq) + S_norm = max(0, qam-qma) · r

Key Lessons

  • Check model config's key_length/value_length first — hidden/num_heads is Q-side; K storage may differ (gated attention, partial rotary)
  • Eviction permanence requires zeroing both Part A and Part B — leaving the scoring path intact lets the slot re-enter selection
  • Attention sink is mandatory, not optional — softmax collapse destroys generation quality immediately (infinite loops)
  • Design APIs as 3-step from the start — upload/reuse/cleanup separation is natural. Starting with a single function incurs big refactoring cost
  • make -j doesn't auto-resolve target dependencies — to re-link binaries, specify target explicitly
  • Audits find new issues even when repeated — self-review has inherent limits. Time gap + different perspective = best substitute
  • Feedback memory actually works — user caught my "No Fallbacks" violation mid-audit and made me revert
